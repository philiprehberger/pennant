## Conventions

- **No AI attribution.** No `Co-Authored-By`, no "generated by Claude," no AI mentions in commits, code, comments, anything.
- **OpenAPI is the source of truth.** Endpoints in `routes/api.php` conform to `openapi/spec.yaml` — not the other way around. CI runs Spectral lint + contract tests. When you add a route, add it to the spec first, then implement the controller to match.
- **MySQL, not Postgres.** No `jsonb`, no `tsvector`, no partial unique indexes. Use `json` columns + application-layer constraints.
- **ULIDs for all IDs**, stored as `CHAR(26)`.
- **`Authorization: Bearer pn_srv_live_…` or `pn_clt_live_…`** for auth. Two key kinds:
  - **Server keys** (`pn_srv_`) — full read of flag rules + evaluation API + audit. Server-side SDKs.
  - **Client keys** (`pn_clt_`) — read-only of pre-evaluated values for a given context. Safe to embed in browser bundles.
  Keys are hashed at rest; `last_four` + `prefix` are stored plaintext for display.
- **RFC 7807 problem details** for error responses (`application/problem+json`).
- **Rule engine drift is the enemy.** Every SDK ships its own evaluator. The `tests/corpus/rules.json` file is the contract; any rule semantics change has to land in the corpus *first*, then in each implementation. Server (PHPUnit) and every SDK (Vitest, etc.) round-trip the corpus on every CI run.
- **No premature abstraction.** Three similar classes beats a generic base class.

## Deployment

- **All halves build locally and rsync to the EC2 host** (54.190.150.0). No CI-hosted builds.
- **Apache + Let's Encrypt** for SSL on `pennant.philiprehberger.com` and `api.pennant.philiprehberger.com`. Vhost edits go in `infra/apache/` *and* on the server — keep them in sync.
- **PM2 port (next free per `/var/www/ecosystem.config.js`)** for the Next.js docs site. The reverse-proxy vhost forwards there.
- **MySQL `pennant` schema** on the EC2 host. The DB password lives in `/var/www/pennant/shared/.env` on the server and in `.local/server-secrets.env` locally (both gitignored).
- **SSE broadcaster** under supervisord — see `infra/sse-broadcaster/` for the Node fan-out worker and `infra/supervisor/pennant-sse.conf` for the supervisor entry.

## Things to avoid

- Don't deploy the API until at least the `/v1/healthz` endpoint returns 200. The deploy script's health check fails the rollout otherwise.
- Don't run `php artisan migrate:fresh` against the production DB without asking.
- Don't change rule semantics without adding corpus rows first. **The corpus is the contract.** If the PHP evaluator and a SDK disagree on a rule, customer experiments silently corrupt — that's the failure mode this discipline exists to prevent.
- Don't write SDK code by hand for endpoint clients. Regenerate from `openapi/spec.yaml`. The rule evaluator + persistence + SSE client are the only hand-written layers.
